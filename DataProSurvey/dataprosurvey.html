Data Professional Survey Breakdown

In a world increasingly powered by data, understanding the professionals behind the insights has never been more critical.

From data engineers to analysts, these roles form the backbone of modern decision-making, using innovative tools and technologies to unlock value from raw information.

This report dives into the perspectives and trends shaping the data profession today, offering a closer look at salary ranges, preferred programming languages, and job satisfaction across different roles.

What are the key factors shaping the career paths of data professionals today?

Ojective

The objective was to analyze survey data from data professionals—including their roles, salaries, preferred tools, and job satisfaction levels—to identify key trends shaping the industry. This analysis aims to provide actionable insights for understanding career dynamics and supporting informed decision-making for both professionals and organizations in the data field.

The dataset

The data used for this project was sourced from Kaggle, a well-known platform for datasets and data science competitions.

The dataset consists of 631 observations and includes 15 qualitative variables. Among these variables are key metrics such as job title, salary, preferred programming language, gender, geographic location, and satisfaction with work-life balance.

Data preparation

To begin with, we simplified the data by removing columns containing too many specific options, including :
“Q1 - Which Title Best Fits your Current Role?”,
“Q4 - What Industry do you work in?”,
“Q5 - Favorite Programming Language?”
“Q11 - Which Country do you live in?.1”.
These columns were cleaned up by grouping all the specific options under the “Other” category, thus optimally cleaning up the data.

Next, we processed the column “Q3 - Current Yearly Salary (in USD)”, which showed ranges of values. For each range, we calculated an average. This was done by duplicating the column and separating the numerical values from the other characters. In this way, we were able to obtain the numerical values for each side and add an average column to represent the mean value for each salary range.
